# 真佛宗法會報導產生器

將法會同修的逐字稿（語音辨識文字）整理為結構化的報導文章，並輸出為格式化的 `.docx` 檔案。

## 工作流程

### 第一步：確認資料

1. 確認目標日期的逐字稿檔案位置（通常在 `{YYYYMMDD}/` 或已命名的資料夾中，檔名如 `{YYYYMMDD}逐字.txt`）
2. 閱讀至少兩份既有報導的 docx 檔案（用 python-docx 解析），掌握最新的格式慣例。格式可能隨時間微調，以最近的報導為準。

### 第二步：閱讀並理解逐字稿

逐字稿是語音辨識的原始輸出，包含大量錯字和不通順的句子。閱讀時需要：

1. **理解整體脈絡**：辨識法會類型（同修、護摩、慶典等）、同修本尊、講經內容
2. **識別段落結構**：哪些部分是迴向、開示、問答、笑話、講經
3. **標記需要修正的錯誤**：參考 `references/terminology.md` 進行術語修正

### 第三步：撰寫報導

根據逐字稿內容，按照以下結構撰寫報導。段落順序應依照法會實際進行的時間順序排列，不同法會的順序可能不同。

#### 報導結構

每個段落之間以分隔符 `🌼 🌼 🌼 🌼 🌼 🌼 🌼` 隔開。

1. **標題** `🧘【同修報導】`
   - 常規同修：`🧘【同修報導】{YYYY}年{M}月{D}日蓮生法王於美國〈西雅圖雷藏寺〉主持「{本尊}本尊法同修」暨講授《大佛頂首楞嚴經》`
   - 特殊法會：依實際內容調整標題

2. **迴向** `💗法王同修後作迴向：`
   - 迴向文必須**完整忠實呈現**，只修正語音辨識的錯字，不可精簡、刪減或改寫任何內容。迴向是法王親口唸誦的祈請文，每一句都有其意義，一字不漏地還原是對法王和迴向對象的尊重。
   - 注意常見錯字：惡運→厄運、惡因→惡緣、法身→化身（視上下文）、鴻胸化肌→逢凶化吉、齋陽→災殃、齋羅生→地羅神、田羅生→天羅神

3. **法王開示** `🍀法王開示`
   - 將口語整理為流暢的書面語，但保留法王的語氣和風格
   - 不要過度修飾，保持開示的親切感

4. **特殊主題段落**（視內容而定）
   - 用適當的 emoji 標題，如 `📖 觀世音菩薩與松贊干布`
   - 當法王深入某個特定主題時，可以獨立成段

5. **法王與弟子問答互動** `📫 法王與弟子問答互動`
   - 問題：`🙋問：`（後接問題內容，另起一段）
   - 回答：`👨‍🏫法王答：`（後接回答內容，另起一段）
   - 多個問答依序排列

6. **法王的幽默** `😂法王的幽默`
   - 從逐字稿中挑選笑話，**最多 3 則**
   - 優先選擇最有意義或最好笑的
   - 每則笑話結尾加 `🤣`

7. **楞嚴經講授** `⚡接着，聖尊講授《大佛頂首楞嚴經》——{主題}：`
   - 原文標題：`📒《大佛頂首楞嚴經》【原文】`
   - 經文引用：以 `♦` 開頭，設為**粗體**
   - 解釋標題：`🔎【解釋】`
   - 解釋內容：以 `⚡` 開頭
   - 經文原文務必從 `doc/大佛頂首楞嚴經.txt` 中用 Grep 搜尋並對照正確文字，不要依賴逐字稿的錯誤文字，也不要憑記憶引用。逐字稿中的經文錯字非常多，必須逐字核對原文。

8. **結咒** `🌸嗡嘛呢唄咪吽！`

9. **結語** `💌 精彩寶貴的開示後，恭請聖尊賜授新皈依灌頂，並加持大悲咒水，及為佛像開光。網路直播告一段落，同修圓滿吉祥！`

10. **來源** `💻 資料畫面來源：西雅圖雷藏寺Youtube直播影片截圖。`

#### 撰寫原則

- **稱呼慣例**：使用「聖尊」「法王」「蓮生法王」「蓮生活佛」「師尊」稱呼根本上師。弟子提問時使用法名（如「蓮花XX」），但若語音辨識的法名無法確認正確寫法，就省略法名，不要猜測。
- **不確定的內容寧可刪除**：如果逐字稿中某段內容無法確認正確意思，或找不到參考資料佐證，寧可移除該段，避免誤導讀者。這原則也適用於人名、法名、地名等專有名詞。
- **笑話取捨**：法王在開示中會穿插多則笑話，報導中精選最多 3 則，選擇標準是最能帶出教義意涵或最具幽默感的。
- **書面語轉換**：將口語轉為書面語時，保留意思的完整性，但去除重複、語助詞、口誤等。

### 第四步：產生 docx 檔案

使用 `scripts/generate_docx.py` 產生格式化的 docx 檔案。呼叫方式：

```bash
python scripts/generate_docx.py --input report_content.json --output "output.docx"
```

或者直接用 python-docx 撰寫生成腳本，格式要求：
- **字型**：DFKai-SB（標楷體）
- **字級**：13pt（165100 EMUs）
- **經文原文**（♦ 開頭的段落）：粗體
- **段落間**：以空白段落分隔
- 所有段落 style 使用 `normal`

### 第五步：重新命名資料夾

將資料夾重新命名為符合慣例的格式：
`{YYYYMMDD}-{報導標題}`

例如：`20260411-蓮生法王於美國〈西雅圖雷藏寺〉主持「觀世音菩薩本尊法同修」暨講授《大佛頂首楞嚴經》`

## 重要提醒

- 語音辨識的逐字稿錯誤非常多，尤其是佛教術語和人名。務必參考 `references/terminology.md` 進行修正。
- 每次工作前，先閱讀最近的既有報導 docx，確認最新的格式慣例。
- 產出報導後，提醒使用者檢查內容是否正確，特別是經文原文和專有名詞。
